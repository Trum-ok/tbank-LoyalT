# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Описание проекта

Платформа управления лояльностью и клиентским опытом для клиентов Т-Банка.

Сервис интегрируется в приложение Т-Банка (раздел «Город») и объединяет клиентов и предпринимателей в единой системе
лояльности: малый бизнес запускает программы вознаграждений, анализирует поведение клиентов и повышает вовлечённость —
клиенты получают бесшовный опыт через T-ID без отдельных приложений и карточек.

### Проблематика

- Удержание клиентов — ключевая задача любого бизнеса.
- Малый бизнес вынужден самостоятельно строить системы лояльности, хотя механик немного и они повторяются.
- Существующие решения (Stamp, Loyverse и др.) требуют отдельного приложения и сложной интеграции.
- Нет удобной встраиваемой системы, нативной для банковской экосистемы.

## Технический стэк

### БД

- **PostgreSQL** — основное хранилище (один инстанс, schema-per-service)
- **Redis** — кэш и сессии (при необходимости)

### Бэкенд

- **Python 3.13+**
- **FastAPI** — обязательно использовать `async`/`await` везде, где это возможно
- **SQLAlchemy** — ORM, драйвер `psycopg`
- **Alembic** — миграции; все изменения схемы только через миграции, не через `CREATE TABLE` вручную

### Фронтенд

- **TypeScript**
- **Angular** (CLI на esbuild, standalone-компоненты, signals по умолчанию)
- **Taiga UI** (`@taiga-ui/core`, `@taiga-ui/kit`, `@taiga-ui/layout`, `@taiga-ui/icons`) — банковский UI-кит Т-Банка
- **pnpm workspaces** — фронт-приложения и общий пакет лежат под `frontend/`

### Брокер сообщений

- **Kafka**

## Архитектура

Система строится как набор микросервисов. Каждый сервис — отдельное FastAPI-приложение со своей областью
ответственности.

### Коммуникация между сервисами

- **Внешние клиенты → сервисы**: через единый API Gateway (маршрутизация, аутентификация, rate limiting)
- **Сервисы → сервисы (синхронно)**: REST через внутреннюю сеть
- **Сервисы → сервисы (асинхронно)**: Kafka

> Правило: если операция не требует немедленного ответа (например, отправка push после начисления баллов) — использовать
> асинхронную очередь, не синхронный вызов.

### Сервисы

| Сервис                 | Ответственность                                                                                                                                                    |
|------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `partner-service`      | ЛК партнёра: регистрация бизнеса, настройка программы лояльности, управление наградами, дашборд аналитики, собственная авторизация                                 |
| `core-service`         | Программы лояльности и правила начисления, каталог наград, начисление/списание баллов, история транзакций, отмена, профиль клиента, баланс, подключённые программы |
| `notification-service` | Push-уведомления: начисление баллов, новые акции, сгорание баллов                                                                                                  |
| `admin-service`        | Панель Т-Банка: модерация заявок партнёров, управление каталогом, блокировка партнёров                                                                             |

## Важные принципы для Claude Code

- **Не создавать новые сервисы** без явного указания — сначала уточни, в какой сервис добавлять код.
- **Не менять схему БД напрямую** — только через Alembic-миграции.
- **Асинхронность обязательна** — все endpoint-хендлеры, обращения к БД и внешним сервисам должны быть `async`.
- **Если архитектурное решение не зафиксировано** (помечено TODO) — явно спроси, не принимай решение самостоятельно.
