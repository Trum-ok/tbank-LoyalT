# Функциональные требования

> **Статусы:** `[x]` — реализовано · `[ ]` — не реализовано · `[~]` — частично
> (см. примечание после пункта). Срез актуален на 2026-05-17, сверен с кодом.

## Самые главные

- [x] Личный кабинет партнера. В рамках ЛК партнер может настраивать программу лояльности.
- [x] Клиент может воспользоваться программой лояльности. Нужно реализовать способ, чтобы показать работоспособность.
- [x] Статистика. В рамках ЛК партнер может увидеть статистику по клиентам.

---

## Что может делать клиент?

### Программы лояльности

- [x] Участвовать в программах лояльности (по QR, ссылке или через каталог в приложении);
- [x] удалять программы у себя;
- [x] архивировать программы у себя (скрыть из основного списка, не удаляя историю) — `enrollment.is_archived`;
- [x] переименовывать программы у себя (отображаемое имя) — `enrollment.display_name`;

### Баллы

- [x] Смотреть баланс баллов по каждой программе;
- [x] смотреть историю начислений и списаний (дата, сумма, описание);
- [x] показывать QR-код на кассе для начисления или списания баллов (`short_code` / lookup);
- [x] копить баллы;
- [x] тратить баллы — выбирать награду из каталога партнёра и получать QR на списание.

### Навигация и каталог

- [x] Смотреть свои активные программы списком / карточками;
- [x] смотреть каталог доступных программ по категориям:
    - еда и напитки (кафе, рестораны, доставка)
    - красота и здоровье (салоны, барбершопы, фитнес)
    - розница (магазины, аптеки)
    - услуги (химчистки, ремонт, автосервис)
    - развлечения (кино, квесты, хобби)
- [x] искать программы по названию бизнеса;

### Уведомления

- [x] Получать push о начислении баллов («+80 бонусов · Кофе Хауз»);
- [x] получать push о доступной награде («Накопили на бесплатный кофе!») — `reward.available`;
- [x] получать push о сгорании баллов («120 баллов сгорят через 3 дня») — `points.expiring`/`points.expired`;
- [ ] получать push о новых акциях от партнёров, чьи программы подключены — событие `partner.new_promotion` никем не публикуется, в notification-service только лог-заглушка.

## Что может делать бизнес? (те, кто создают партнерские программы)

### Онбординг

- [x] Подать заявку на регистрацию: название, категория, ИНН, контакт;
- [x] ждать модерацию от Т-Банка (статус заявки виден в ЛК);
- [x] после одобрения — получить доступ к ЛК партнёра;
- [x] заводить кассиров / сотрудников и выдавать им доступ к кассовому приложению по JWT *(сверх исходных требований)*;

### Программа лояльности (no-code конструктор)

- [x] Выбрать шаблон программы:
    - Накопительная (баллы за каждые N рублей)
    - За визит (фиксированные баллы за каждое посещение)
    - Штампы (X визитов → награда, аналог карточки с дырочками)
- настроить правила начисления:
    - [x] Процент от суммы покупки (например, 5% → 5 баллов за 100 ₽)
    - [x] Фиксированная сумма за визит
    - [ ] Повышенные баллы в определённые дни / часы («двойные баллы по средам») — есть только множитель по уровню (`tier`), правил по дню недели/часу нет
- настроить награды:
    - [x] Скидка в % или фиксированная сумма (`discount_percent` / `discount_fixed`)
    - [x] Бесплатный товар / услуга (`free_item`)
    - [x] Повышенный кэшбэк (`cashback_boost`)
- [x] задать срок действия баллов (30 / 60 / 90 дней / бессрочно);
- [x] задать минимальный порог списания;
- [x] загрузить логотип и выбрать цвет (брендинг карточки клиента);
- [x] опубликовать / приостановить / архивировать программу;
- [x] настроить уровни программы (tiers): пороги баллов с множителем начисления *(сверх исходных требований)*;
- [x] задать лимиты программы: минимальная сумма покупки, максимум баллов за транзакцию, максимальный % списания, окно действия `valid_from`/`valid_until` *(сверх исходных требований)*.

### Геймификация

- Настроить бонусные акции:
    - [ ] «Купи X штук товара Y, получи Z баллов»
    - [ ] «Приведи друга — оба получают бонус» — поле `referral_bonus_points` есть, логика начисления не вызывается
    - [ ] «Streak: посети нас 5 раз подряд — двойные баллы»
    - [ ] «День рождения: +N баллов в день рождения клиента» — поля `birthday_bonus_*` есть, логика начисления не вызывается
- [ ] задать период действия акции (дата начала и конца) — сущности «акция» нет (у программы есть `valid_from/until`, но не у акций);
- [ ] совмещать несколько акций одновременно.
- [x] Welcome-бонус: +N баллов при вступлении в программу (`welcome_bonus_points`) *(сверх исходных требований)*.

### Коммуникации

- [x] Отправить push-уведомление всем участникам программы;
- [x] отправить push сегменту (`all_enrolled` / `by_program` / `balance_positive` / `new_7d` / `active_30d`);
- [x] просмотреть историю отправленных уведомлений.

### Статистика и аналитика

- [x] Дашборд: активные клиенты, выдано баллов, списано баллов, средний чек;
- [x] график новых участников программы по дням / неделям;
- [x] Retention rate: % клиентов, вернувшихся в течение 30 дней (d1/d7/d30 + кривая);
- [x] топ дней недели по активности (heatmap день×час);
- [x] таблица последних транзакций;
- [x] stickiness: DAU / WAU / MAU и их отношения *(сверх исходных требований)*;
- [ ] RFM-сегментация: давность / частота / сумма покупок клиентов;
- [ ] AI-инсайты: автоматические рекомендации на основе данных
  (пример: «38% клиентов не вернулись после первого визита — запустите акцию для новичков»);
- [ ] экспорт статистики в CSV;

## Что может делать владелец платформы? (Т-банк, админка)

### Модерация

- [x] Смотреть очередь заявок от бизнесов на подключение;
- [x] одобрить / отклонить заявку с комментарием;
- [x] просматривать профиль бизнеса (ИНН, категория, контакты);
- [x] приостановить или заблокировать действующего партнёра.

### Метрики платформы

- [x] Общее число партнёров (активных / на модерации / заблокированных);
- [x] общее число клиентов на платформе;
- [x] количество транзакций за период (начисления / списания);
- [ ] оборот баллов в рублёвом эквиваленте;
- [x] топ партнёров по активности клиентов;
- [x] новые регистрации (партнёры и клиенты) по дням;
- [ ] retention клиентов по платформе в целом (есть только на уровне партнёра в core-аналитике);
- [ ] доля и количество платных клиентов (бизнесов) — биллинга нет.

### Управление каталогом

- [~] Редактировать категории каталога — доступны переопределения (label, описание, порядок, активность), но категории — фиксированный enum: «добавить / переименовать / удалить» новую категорию нельзя;
- [x] выделять партнёров в «Рекомендованные» (отображаются первыми в каталоге) — фича «платная» по смыслу, биллинг не реализован;
- [x] управлять баннерами и промо-блоками в каталоге клиента.

---

*Технически (вне функционального перечня) реализованы: сквозной `X-Request-ID`,
read-реплика для тяжёлых выборок, HASH-партиционирование транзакций,
идемпотентная проекция аналитики.*
